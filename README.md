## نظرة عامة

**البنية:**
- **جهاز السيرفر**: يشغل Backend + قاعدة البيانات (يكون على أحد الجهازين - يُفضل جهاز السكرتير لأنه يعمل طوال اليوم)
- **جهاز السكرتير**: يفتح المتصفح على `/secretary` لإدخال المرضى
- **جهاز الدكتورة**: يفتح المتصفح على `/doctor` لفحص المرضى
- **قاعدة البيانات**: ملف واحد `clinic.db` على جهاز السيرفر

**الأمان:**
- رمز دخول موحد (PIN) للعيادة
- كل الطلبات محمية برمز الدخول
- Rate limiting لمنع الهجمات
- CORS مقيد

## الجزء 1: إعداد جهاز السيرفر (جهاز السكرتير أو الدكتورة)

### الخطوة 1: تحميل المشروع

انسخ مجلد المشروع كاملاً إلى الجهاز، مثلاً في:
```
C:\ClinicApp\
```

الهيكل المتوقع:
```
ClinicApp/
├── backend/
│   ├── server.py
│   ├── requirements.txt
│   └── .env
├── frontend/
│   ├── src/
│   ├── package.json
│   └── .env
└── README.md
```

### الخطوة 2: تثبيت Backend

افتح **Command Prompt** (CMD) وشغّل:

```bash
cd C:\ClinicApp\backend
pip install -r requirements.txt
```

### الخطوة 3: إعداد ملف Backend .env

افتح ملف `backend/.env` وعدّله ليصبح كالتالي:

```env
DATABASE_PATH=C:\ClinicApp\backend\clinic.db
CLINIC_PIN=1234
CORS_ORIGINS=http://localhost:3000,http://192.168.1.100:3000
```

> **مهم جداً**:
> - غيّر `CLINIC_PIN` إلى رمز سري خاص بك (مثلاً `7823`)
> - غيّر `192.168.1.100` إلى IP جهاز السيرفر الحقيقي (سنشرح كيف تحصل عليه في الخطوة 6)

### الخطوة 4: تثبيت Frontend

في نافذة CMD جديدة:

```bash
cd C:\ClinicApp\frontend
yarn install
```

### الخطوة 5: معرفة IP جهاز السيرفر

في CMD:
```bash
ipconfig
```

ابحث عن السطر:
```
IPv4 Address. . . . . . . . . . . : 192.168.1.100
```

**احفظ هذا الرقم** (سيكون مثل `192.168.1.100` أو `192.168.0.5`).

### الخطوة 6: إعداد ملف Frontend .env

افتح ملف `frontend/.env` وضع فيه:

```env
REACT_APP_BACKEND_URL=http://192.168.1.100:8001
```

> استبدل `192.168.1.100` بـ IP الذي حصلت عليه في الخطوة 5.

### الخطوة 7: تشغيل التطبيق

**افتح نافذتين من CMD:**

**النافذة الأولى (Backend):**
```bash
cd C:\ClinicApp\backend
uvicorn server:app --host 0.0.0.0 --port 8001
```

**النافذة الثانية (Frontend):**
```bash
cd C:\ClinicApp\frontend
yarn start
```

سيفتح المتصفح تلقائياً على `http://localhost:3000`.

### الخطوة 8: السماح للجدار الناري (Firewall)

على ويندوز:
1. اذهب إلى **Control Panel > Windows Defender Firewall > Advanced Settings**
2. اختر **Inbound Rules > New Rule**
3. اختر **Port > TCP** وأدخل `8001,3000`
4. اختر **Allow the connection**
5. أطلق الاسم "Clinic App"

## الجزء 2: ربط الجهاز الثاني (عبر الشبكة)

**على الجهاز الثاني (سكرتير أو دكتورة):**

1. تأكد أنه متصل بنفس شبكة WiFi/LAN
2. افتح المتصفح (Chrome/Edge)
3. للسكرتير، افتح:
   ```
   http://192.168.1.100:3000/secretary
   ```
4. للدكتورة، افتح:
   ```
   http://192.168.1.100:3000/doctor
   ```
5. أدخل رمز الدخول (`CLINIC_PIN` الذي وضعته)

**نصيحة**: احفظ الرابط في المفضلة (Bookmark) على كل جهاز.

## الجزء 3: النسخ الاحتياطي عبر Google Drive

### الطريقة البسيطة (بدون كود):

**الخطوة 1**: ثبّت برنامج Google Drive for Desktop على جهاز السيرفر:
- [تحميل من هنا](https://www.google.com/drive/download/)

**الخطوة 2**: سجّل الدخول بحساب Google الخاص بالعيادة

**الخطوة 3**: أنشئ مجلداً داخل Google Drive:
```
My Drive/Al-Siraj Backup/
```

**الخطوة 4**: أوقف Backend مؤقتاً (Ctrl+C في نافذة uvicorn)

**الخطوة 5**: انقل ملف `clinic.db` من `C:\ClinicApp\backend\` إلى مجلد Google Drive:
```
C:\Users\<username>\My Drive\Al-Siraj Backup\clinic.db
```

**الخطوة 6**: عدّل `backend/.env`:
```env
DATABASE_PATH=C:\Users\<username>\My Drive\Al-Siraj Backup\clinic.db
```

**الخطوة 7**: أعد تشغيل Backend

**النتيجة**: Google Drive سيزامن الملف تلقائياً مع كل تحديث! يمكنك رؤية النسخ الاحتياطية عبر تاريخ الإصدارات في Drive.

## الجزء 4: تثبيت التطبيق كـ PWA

### على الويندوز (Chrome/Edge):
1. افتح التطبيق في المتصفح
2. اضغط على أيقونة التثبيت (⊕) في شريط العنوان (يمين URL)
3. اختر **"تثبيت"**
4. سيظهر التطبيق كأيقونة على سطح المكتب

### على الهاتف (Android):
1. افتح التطبيق في Chrome
2. من القائمة (⋮) اختر **"إضافة إلى الشاشة الرئيسية"**
3. سيظهر كتطبيق مستقل

**الفائدة**: التطبيق يعمل كتطبيق حقيقي بدون شريط عنوان، وأسرع في الوصول.

## الجزء 5: الأمان والخصوصية

### الحماية المطبقة:
1. **رمز دخول موحد** (PIN) - كل الوصول محمي به
2. **Rate Limiting** - يمنع الهجمات (10 محاولات دخول/دقيقة)
3. **CORS مقيد** - يقبل فقط الأجهزة المحددة
4. **حدود إدخال** - يحمي من الإدخال الضار
5. **جلسة محلية** - رمز الدخول يُحفظ في المتصفح فقط

### نصائح مهمة:
- ⚠️ **غيّر PIN الافتراضي** (`1234`) إلى رمز سري بمجرد التثبيت
- 🔒 **لا تشارك PIN** إلا مع الطبيبة والسكرتير
- 💾 **خذ نسخة احتياطية يدوية** من `clinic.db` أسبوعياً (بالإضافة لـ Google Drive)
- 🌐 **لا تفتح المنافذ (3000، 8001) على الإنترنت** - فقط الشبكة المحلية

### تغيير رمز الدخول (PIN):
1. أوقف Backend
2. عدّل `CLINIC_PIN` في `backend/.env`
3. أعد تشغيل Backend
4. اطلب من الجميع تسجيل الخروج والدخول مجدداً بالرمز الجديد

## حل المشاكل الشائعة

### المشكلة: لا يمكن الوصول من الجهاز الثاني
**الحل:**
- تأكد أن كلا الجهازين على نفس الشبكة (WiFi)
- تأكد من IP الصحيح في `frontend/.env`
- تأكد من تفعيل قواعد Firewall (الخطوة 8)
- جرب من CMD: `ping 192.168.1.100`

### المشكلة: "رمز الدخول غير صحيح"
**الحل:**
- تأكد من مطابقة PIN في `backend/.env` مع ما تُدخله
- امسح ذاكرة المتصفح (localStorage)
- أعد تشغيل Backend بعد أي تغيير في `.env`

### المشكلة: قاعدة البيانات فارغة بعد نقل الملف
**الحل:**
- تأكد أن `DATABASE_PATH` في `.env` يشير للمسار الصحيح
- تأكد من صلاحيات القراءة/الكتابة على الملف
- افتح الملف ببرنامج [DB Browser for SQLite](https://sqlitebrowser.org/) للتحقق

### المشكلة: Backend لا يبدأ
**الحل:**
- تأكد من تثبيت جميع المكتبات: `pip install -r requirements.txt`
- تأكد أن المنفذ 8001 غير مستخدم من برنامج آخر
- اقرأ رسالة الخطأ في CMD

### المشكلة: البيانات لا تظهر في واجهة الدكتورة
**الحل:**
- اضغط زر Refresh (🔄) في قائمة المرضى
- تأكد أن الاتصال بالسيرفر يعمل (افحص Console في المتصفح: F12)

## معلومات التطبيق

**عيادة السراج لطب العيون**
الطبيبة الاستشارية: د.وسن عبد العزيز رشيد

**التقنيات المستخدمة:**
- Backend: Python + FastAPI + SQLite
- Frontend: React + Tailwind CSS + Shadcn UI
- PWA: Service Worker + Web Manifest

**نظرة سريعة على الواجهات:**
- `/login` - صفحة تسجيل الدخول
- `/secretary` - واجهة السكرتير
- `/doctor` - واجهة الطبيبة

**API Endpoints:**
- `POST /api/patients` - إنشاء مريض
- `GET /api/patients` - قائمة المرضى (مع فلترة/بحث)
- `PUT /api/patients/{id}` - تحديث فحص
- `DELETE /api/patients/{id}` - حذف مريض
- `GET /api/shortcuts` - قائمة الاختصارات
- `POST /api/shortcuts` - إضافة اختصار
- `PUT /api/shortcuts/{id}` - تعديل اختصار
- `DELETE /api/shortcuts/{id}` - حذف اختصار
- `GET /api/stats` - إحصائيات العيادة
