# نشر عيادة السراج على PythonAnywhere (مجاني بالكامل، بدون بطاقة ائتمان)

هذا الدليل يشرح خطوة بخطوة كيف تنشر التطبيق كموقع دائم يعمل بدون أي جهاز محلي،
وبدون أي بطاقة ائتمان، على استضافة **PythonAnywhere** المجانية.

> **ملاحظة مهمة قبل البدء:** لأن الخطة المجانية من PythonAnywhere لا تدعم
> WebSocket، تم تعديل آلية التحديث اللحظي في التطبيق لتعمل عبر **Polling**
> (سؤال دوري كل 3-5 ثوانٍ) بدل الدفع الفوري. الفرق العملي بسيط جداً (تأخير
> بضع ثوانٍ فقط) وكل الميزات الأخرى تعمل تماماً كما هي.

---

## 1. إنشاء الحساب

1. اذهب إلى: https://www.pythonanywhere.com/registration/register/beginner/
2. سجّل باستخدام بريد إلكتروني فقط — **لا يُطلب أي بطاقة ائتمان أو دفع إطلاقاً**.
3. بعد التسجيل، سجّل دخولك للوحة التحكم.

---

## 2. رفع الكود

الطريقة الأسهل: من تبويبة **Consoles** بلوحة تحكم PythonAnywhere، افتح **Bash console** جديدة، ثم:

```bash
git clone https://github.com/doseo8808/clinic-app.git
```

(هذا يفترض أنك رفعت الكود المُحدَّث فعلاً على مستودعك بـ GitHub — إذا لم تفعل بعد، ارفعه أولاً بنفس الخطوات اللي استخدمناها سابقاً.)

---

## 3. إنشاء بيئة Python وتثبيت المكتبات

بنفس الـ Bash console:

```bash
mkvirtualenv --python=/usr/bin/python3.10 clinic-env
cd clinic-app/backend
pip install -r requirements-deploy.txt
```

انتظر حتى ينتهي التثبيت (دقيقة أو دقيقتين).

---

## 4. إعداد رموز PIN

بنفس الـ console:

```bash
nano .env
```

اكتب بداخله (غيّر الأرقام لأي رمز تختاره أنت):

```
SECRETARY_PIN=1357
DOCTOR_PIN=9642
```

احفظ بالضغط على `Ctrl+O` ثم `Enter`، واخرج بـ `Ctrl+X`.

⚠️ **مهم:** غيّر الرمزين لغير `1234`/`4321` الافتراضيين قبل ما يصبح الموقع متاحاً على الإنترنت العام.

---

## 5. إنشاء تطبيق الويب (Web App)

1. من لوحة التحكم اذهب إلى تبويبة **Web**.
2. اضغط **Add a new web app**.
3. اختر النطاق المجاني المقترح (مثل `yourusername.pythonanywhere.com`).
4. عند سؤالك عن نوع الإعداد، اختر **Manual configuration** (وليس Django/Flask الجاهز).
5. اختر إصدار Python **3.10**.

---

## 6. ربط بيئة Python (Virtualenv)

في نفس صفحة الـ Web app، ابحث عن قسم **Virtualenv**، واكتب المسار:

```
/home/yourusername/.virtualenvs/clinic-env
```

(استبدل `yourusername` باسم المستخدم الفعلي عندك بـ PythonAnywhere)

---

## 7. تعديل ملف WSGI

في نفس الصفحة، ابحث عن رابط باسم شبيه بـ:
`/var/www/yourusername_pythonanywhere_com_wsgi.py`

اضغط عليه لفتحه بالمحرر، **احذف كل المحتوى الموجود** واستبدله بهذا بالضبط (غيّر `yourusername` فقط):

```python
import sys

project_home = '/home/yourusername/clinic-app/backend'
if project_home not in sys.path:
    sys.path.insert(0, project_home)

from wsgi import application
```

احفظ الملف.

---

## 8. الواجهة الأمامية (React) — جاهزة مسبقاً ✅

النسخة الموجودة معك بالفعل تحتوي على مجلد `backend/frontend_build` مبنياً وجاهزاً — لا تحتاج تسوي أي شيء بهذه الخطوة، فقط تأكد عند رفع الكود على GitHub أن هذا المجلد مرفوع معه (تحقق أنه غير مُستثنى بملف `.gitignore`).

**فقط إذا عدّلت الكود لاحقاً** (مثلاً أضفت ميزة جديدة) وتحتاج تبني نسخة محدّثة، على جهاز فيه Node.js:

```bash
cd frontend
yarn install
yarn build
```

ثم انسخ محتوى `frontend/build` إلى `backend/frontend_build` وارفعه على GitHub وأعد `git pull` على PythonAnywhere.

---

## 9. التشغيل

ارجع لتبويبة **Web** واضغط الزر الأخضر **Reload**.

بعدها افتح رابط موقعك: `https://yourusername.pythonanywhere.com` — يجب أن تظهر صفحة تسجيل الدخول مباشرة.

---

## 10. اختبار قبل الاستخدام الفعلي

- افتح الموقع على جهازين مختلفين بنفس الوقت (متصفحين، أو جهاز وهاتف).
- سجّل مريضاً من شاشة السكرتير، وتأكد أنه يظهر عند الدكتورة خلال ثوانٍ قليلة (Polling).
- جرّب فتح نفس المريض من الجهازين وتعديل حقل، وتأكد أن التحديث ينتقل (بتأخير بسيط طبيعي).

## ملاحظات مهمة

| الموضوع | التفاصيل |
